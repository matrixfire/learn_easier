# Learn_Easier Note Standards

## Principles

1. **Scannable & Natural** - List-based hierarchy, easy to scroll up/down
2. **Concise & Focused** - LLM processes to condense lengthy content
3. **Minimalist & Clean** - No distractions, no excessive formatting
4. **Clear Structure** - Visual hierarchy through indentation

## Format Rules

### Heading Hierarchy
- `#` Main topic / Section
- `##` Subsections
- `###` Detailed points

### List Types
- `-` Bullet points for items
- `- [ ]` Checkboxes for to-do/action items
- `1.` Numbered lists for sequences/steps
- `>` Blockquotes for key insights or definitions

### Content Guidelines
- **Length per bullet:** 1-2 lines max
- **Sentence structure:** Short, active voice
- **Keywords:** Bold `**` for emphasis sparingly
- **Code:** Inline `code` or ```blocks``` when needed
- **Links:** `[text](url)` minimal use

### LLM Processing Goals
- Summarize verbose explanations
- Extract key concepts
- Convert paragraphs → lists
- Remove redundancy
- Preserve context and meaning

## Template Example

```
# Topic Name

## Core Concepts
- Concept 1: brief definition
- Concept 2: what it does

## Key Points
- Point about X
- Point about Y

## Notes/Insights
- Personal observation
- Connection to other topics
```

## Anti-Patterns to Avoid
- ❌ Overly decorative headers/mojis (unless user requests)
- ❌ Dense paragraphs (>3 lines)
- ❌ Multiple bold/italic in one line
- ❌ Excessive nesting (>4 levels)
- ❌ Ambiguous headings

## Ideal Pattern
- ✅ Clean headers
- ✅ 1-2 line bullets
- ✅ Clear hierarchy
- ✅ Natural scrolling flow
