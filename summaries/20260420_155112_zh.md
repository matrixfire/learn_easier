# 编程代理

## 定义
编程代理是具有工具访问权限的对话式 AI 模型：
- 读取/写入文件
- 网络搜索
- 执行 shell 命令

可在 IDE、CLI 或 GUI 工具中使用 - 高度自主且强大

## AI 模型工作原理

### LLM 基础
- LLM 模型化在给定提示字符串下的完成字符串概率分布
- 固定上下文窗口（最大输入/输出长度）
- 推理从此概率分布中采样

### 多轮交互
- 聊天应用程序提供完整对话历史作为提示
- 每个用户提示执行一次 LLM 推理
- 轮次标记分隔对话轮次

### 工具调用代理
- 演绎架构解释特定 LLM 输出为工具调用请求
- 工具结果作为提示字符串的一部分反馈给模型
- 每次工具调用后 LLM 推理再次执行
- 核心概念可用约 200 行代码实现

## 隐私考虑
- 大多数 AI 工具将数据传输到云端
- 部分服务实际上会取得整个存储库的副本
- 开源工具和 LLM 可用但能力较弱
- 由于硬件限制，在本地执行最前沿的开源 LLM 通常不可行

## 使用案例

### 实现新功能
- 提供足够描述性的规格
- 避免过度描述
- 测试驱动开发特别有效：
  - 编写测试（或请代理协助）
  - 确认测试捕获需求
  - 请求代理实现功能

### 修正错误
- 提示代理修正编译器/linter/类型检查器/测试错误
- 在反馈循环中最有效
- 设定模型直接执行失败的检查
- 否则提供手动反馈

### 重构
- 简单：重命名方法
- 复杂：将功能分解到独立模块

### 代码审查
- 审查最新未提交的变更
- 审查 pull request（若代理支持网络获取或 GitHub CLI）

### 代码理解
- 询问关于代码库的问题
- 特别有助于入门

### Shell 任务
- 使用自然语言执行 shell 命令
- 示例：「使用 find 命令查找所有超过 30 天的文件」

### Vibe Coding
- 不编写任何代码实现应用程序

## 高级功能

### 可重复使用的提示
- 为特定工作流程创建详细提示
- 保存为模板以供重复使用

### 并行代理
- 同时执行多个代理副本
- 相同任务：采用最佳解决方案（LLM 是随机的）
- 不同任务：同时实现不重叠的功能
- 使用 git worktree 防止干扰

### MCP (模型上下文通讯协议)
- 开放协议，用于连接代理与工具
- 示例：Notion MCP 服务器用于读取/写入文档
- 通过 Pulse 和 Glama 等目录发现 MCP

### 上下文管理
- 平衡提供必要信息与避免臃肿
- 上下文溢出会降低模型性能
- 代理演绎架构自动提供部分上下文
- 用户维持重要控制权

#### 上下文控制
- **清除**：为无关查询开始新对话
- **回退**：改变对话历史中的步骤而非引导
- **压缩**：对话历史过长时自动摘要前缀
- **llms.txt**：LLM 专用文档的标准位置
- **AGENTS.md/CLAUDE.md**：编程代理的 README，自动加载
- **技能**：避免上下文臃肿的间接层，按需加载
- **子代理**：任务特定的工作流程，隔离上下文

### 子代理
- 顶层代理调用子代理完成特定任务
- 每个代理独立管理自己的上下文
- 示例：网络研究作为子代理
- 可使用 LLM 定义子代理

## 注意事项

### 限制
- AI 工具会犯错（概率模型）
- 不像人类那样的「智慧」
- 检视输出的正确性和安全漏洞
- 验证代码可能比亲自编写更困难
- 关键代码：考虑亲自编写

### 常见陷阱
- 钻牛角尖和误导
- 除错循环
- 过度依赖和浅层理解
- AI 仍无法执行的大量任务类别

### 最佳实践
- 将 AI 作为工具而非拐杖
- 运算思维仍有价值
- 维持对代码的深入理解
