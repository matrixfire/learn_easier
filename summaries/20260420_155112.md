# Agentic Coding

## Definition
Coding agents are conversational AI models with access to tools:
- Reading/writing files
- Web search
- Invoking shell commands

Available in IDEs, CLI, or GUI tools - highly autonomous and powerful

## How AI Models Work

### LLM Basics
- LLMs model probability distribution of completion strings given prompt strings
- Fixed context window (maximum input/output length)
- Inference samples from this distribution

### Multi-Turn Interaction
- Chat apps supply entire conversation history as prompt
- One LLM inference per user prompt
- Turn markers separate conversation turns

### Tool-Calling Agents
- Harness interprets LLM outputs as tool invocation requests
- Tool results fed back to model as part of prompt
- LLM inference runs again after each tool call
- Core concepts can be implemented in ~200 lines of code

## Privacy Considerations
- Most AI tools send data to cloud
- Some services effectively get copy of entire repository
- Open-source tools and LLMs available but less capable
- Running bleeding-edge open LLMs locally often infeasible due to hardware

## Use Cases

### Implementing New Features
- Give descriptive enough specification
- Avoid being overly descriptive
- Test-driven development particularly effective:
  - Write tests (or ask agent to help)
  - Audit tests capture requirements
  - Ask agent to implement feature

### Fixing Errors
- Prompt agent to fix compiler/linter/type checker/test errors
- Most effective in feedback loop
- Set up model to run failing checks directly
- Otherwise provide manual feedback

### Refactoring
- Simple: renaming methods
- Complex: breaking functionality into separate modules

### Code Review
- Review latest uncommitted changes
- Review pull requests if agent supports web fetch or GitHub CLI

### Code Understanding
- Ask questions about codebase
- Particularly helpful for onboarding

### Shell Tasks
- Invoke shell commands using natural language
- Example: "use find command to find all files older than 30 days"

### Vibe Coding
- Implement applications without writing code yourself

## Advanced Features

### Reusable Prompts
- Create detailed prompts for specific workflows
- Save as templates for reuse

### Parallel Agents
- Run multiple copies of agents simultaneously
- Same task: take best solution (LLMs are stochastic)
- Different tasks: implement non-overlapping features in parallel
- Use git worktrees to prevent interference

### MCP (Model Context Protocol)
- Open protocol to connect agents with tools
- Example: Notion MCP server for reading/writing docs
- Discover MCPs via directories like Pulse and Glama

### Context Management
- Balance between providing necessary information and avoiding bloat
- Context overflow degrades model performance
- Agent harnesses automatically supply some context
- User maintains significant control

#### Context Controls
- **Clearing**: Start new conversation for unrelated queries
- **Rewinding**: Undo conversation steps instead of steering
- **Compaction**: Automatically summarize conversation prefix when too long
- **llms.txt**: Standard location for LLM-specific documentation
- **AGENTS.md/CLAUDE.md**: README for coding agents, auto-loaded
- **Skills**: Indirection to avoid context bloat, load on demand
- **Subagents**: Task-specific workflows, isolate context

### Subagents
- Top-level agent invokes subagent for specific tasks
- Each agent manages its own context independently
- Example: Web research as subagent
- Can define subagents using LLMs

## What to Watch Out For

### Limitations
- AI tools make mistakes (probabilistic models)
- Not "intelligent" like humans
- Review output for correctness and security bugs
- Verifying code can be harder than writing it
- Critical code: consider writing by hand

### Common Pitfalls
- Rabbit holes and gaslighting
- Debugging spirals
- Overreliance and shallow understanding
- Large class of tasks AI still incapable of

### Best Practices
- Use AI as tool, not crutch
- Computational thinking still valuable
- Maintain deep understanding of your code
